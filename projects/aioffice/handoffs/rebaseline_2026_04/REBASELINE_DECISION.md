# Rebaseline Decision

Decision status:
- frozen handoff decision set for the new clean repo
- this file is authoritative for the reset direction

## Frozen Decisions
- `V1` is admin-only and self-build first.
- The first acceptable proof boundary is narrow supervised operation through `architect` plus bounded `apply` or promotion control.
- No migration of legacy tasks, milestones, or kanban history is allowed into the new clean repo.
- Preserve doctrine, naming discipline, useful governance and control patterns, and proven process rules.
- Broad UI or control-room work is deferred.
- Git and persisted state remain truth substrates.
- A cleaner rebuild is preferred over preserving noisy history.
- The old repo remains donor or reference material only, not the active rewrite target.

## Consequences For The New Repo
- The new repo starts from a clean constitutional baseline, not from old operational history.
- Product breadth may remain visible in the new vision, but current-proof honesty must remain stricter than product ambition.
- Donor code and donor docs may be reused selectively only where they strengthen the clean reset.
- The reset should preserve the good control model while dropping accumulated milestone narration and historical baggage.

## Explicit Non-Claims
- This decision does not claim the old repo failed as evidence.
- This decision does not claim the broader product vision is already implemented.
- This decision does not authorize a broad UI build, broad later-lane workflow proof, or legacy-backlog migration as part of the reset.
