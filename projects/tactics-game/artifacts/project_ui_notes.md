# Project Ui Notes

## UI Concepts
- The UI should embody a clean, minimalistic design that reinforces the tactical and strategic nature of the game.
- Visual hierarchy must clearly distinguish between active units, available actions, and environmental elements to support player decision-making.
- Controls should be intuitive and responsive, providing smooth toggling between unit activation and movement phases.
- The interface layout should maintain unobstructed views of the isometric battlefield while offering contextual overlays for detailed unit information and current turn indicators.
- Consistent iconography and color coding will reinforce unit statuses, factions, and actionable options.

## Visual Direction
- Use a muted, naturalistic color palette with accent colors reserved for highlights such as active units, threat zones, and spell effects to avoid visual clutter.
- The isometric board and units should have clean, crisp edges with subtle shading to enhance depth perception without overpowering the UI.
- Spellcasting and ability effects should employ distinctive VFX that are bright and animated, but short in duration, to provide clear feedback without disrupting tactical flow.
- Font choices should prioritize readability with clear, generously spaced typefaces in UI panels and tooltips.
- Animations for unit activation and state changes should feel fluid and deliberate to signal tactical shifts.

## Player Feedback
- Spellcasting feedback must include immediate visual confirmation on targets, such as glow outlines and particle bursts, accompanied by a brief UI text cue indicating ability name and effect.
- Successful actions should produce satisfying visual and auditory cues, reinforcing player choices and maintaining engagement.
- Readability is enhanced by dynamically updating HUD elements that summarize unit health, facing, and status effects in an accessible format.
- Turn transitions and key events should trigger subtle screen effects (e.g., light pulses or vignette shifts) to keep players oriented in the flow of combat.
- Tooltips and contextual help should be easily accessible but unobtrusive, supporting newcomers without distracting experienced players.
