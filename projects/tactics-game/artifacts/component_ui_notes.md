# Component UI Notes

## UI Concepts
- **Component Identity:** Use distinct icons and color coding to represent different Component types, ensuring quick recognition.
- **Spellcasting Feedback:** Implement dynamic visual cues such as glowing outlines, casting animations, and progress bars to indicate active spellcasting.
- **Player Readability:** Maintain clean typography with high contrast and concise labels; tooltips provide additional details on hover or focus.

## Visual Direction
- Utilize a consistent isometric style aligned with the game's art direction.
- Employ vibrant but harmonious colors for Components to stand out against the board.
- Animations should be smooth and subtle to avoid distraction but clearly communicate state changes.

## Player Feedback
- Immediate visual response on Component selection, such as highlighting or scaling.
- Clear indicators for spellcasting phases: initiation, channeling, and completion.
- Use sound cues complementing visual feedback to reinforce player actions and game events.
