# Evidence UI Notes

## UI Concepts

- **Evidence Identity:** Use distinct icons and color coding to represent different types of Evidence. Each Evidence item should have a unique symbol and a consistent color palette to aid quick recognition.
- **Spellcasting Feedback:** When Evidence is involved in spellcasting, provide dynamic visual cues such as glowing outlines, pulsing effects, or subtle animations to indicate activation.
- **Player Readability:** Display Evidence information with clear typography and concise text. Tooltips or expandable panels can provide additional details without cluttering the main UI.

## Visual Direction

- Employ a clean, minimalistic style with high contrast between Evidence elements and the background.
- Use a consistent color scheme aligned with the game's overall palette, ensuring Evidence stands out but remains harmonious.
- Animations for spellcasting feedback should be smooth and not overly distracting, emphasizing clarity.

## Player Feedback

- Immediate visual response when Evidence is selected or activated (e.g., highlight, glow).
- Clear indication of Evidence status (active, inactive, cooldown) through color shifts or icon changes.
- Provide subtle sound cues synchronized with visual effects to reinforce player actions involving Evidence.
